# 《Rose Infinity · 玫瑰无限》声音与角色配音方案

> 版本：v1.0  
> 更新日期：2026-07-15  
> 适用范围：网页游戏正式 Demo、比赛录屏与后续公开发布  
> 核心目标：让声音服务于“先亲历，再看见”的叙事；在可落地的前提下尽量降低版权风险。

---

## 1. 结论先行

本作采用三层声音结构：

1. **原创配乐**：5 首母带覆盖标题、甜蜜、裂变、分手、治愈；回看段使用同一主题的变奏，不采购来历不明的“免费 BGM”。
2. **环境与交互音**：优先自行录制或合成；来不及的部分只使用可核验的 CC0 素材，并建立资产登记表。
3. **角色配音**：继续使用项目已有的腾讯云 TTS 离线生成管线；Vera、Sean 和旁白使用三套固定音色，先完成 12 句试音，再批量生成。

不建议把每一行旁白都朗读出来。推荐“**角色全配 + 关键旁白配 + 普通叙述静默**”，既保留视觉小说的阅读节奏，也能让重要台词真正被听见。

## 2. 当前项目声音现状

项目已经具备以下基础：

- `scripts/gen-voice.mts`：遍历 `lib/story.ts`，调用腾讯云 `TextToVoice`，离线生成 MP3。
- `lib/voice-core.ts`：按“音色 + 台词”生成稳定哈希，文本修改后只补生成新音频。
- `lib/use-voice.ts`：客户端按清单播放台词，支持自动停止上一句和持久化静音。
- `app/game/page.tsx`、`app/look/page.tsx`：已经接入台词播放。
- `public/audio/manifest.json`：目前为空，说明正式音频尚未生成。

| 旁白 | 255 | 6,977 |
| Sean | 74 | 1,584 |

| **合计** | **383** | **9,251** |

## 3. 总体声音风格

### 3.1 关键词

水彩、柔光、青春、克制、近距离呼吸感、暖冷对照、留白、微忧郁、最终治愈。

### 3.2 音乐语言

- 核心乐器：毡音钢琴、柔和合成器 Pad、极轻的原声拨弦、低频脉冲。
- 禁止使用：宏大弦乐、煽情哭腔、密集鼓点、短视频“催泪神曲”套路、带歌词的人声歌曲。
- 旋律只保留一个简短的“玫瑰动机”：前半段没有完全落稳，结尾才回到稳定和弦。
- 甜蜜段不是纯糖，要留一丝空；冷段不是恐怖，要让玩家仍然感到“两个人都值得心疼”。
- 音乐不替角色哭。分手最痛的地方优先用静默，而不是把音量推满。

### 3.3 动态原则

- 玩家阅读时，BGM 只负责托住情绪，不抢文字。
- 角色开口时，BGM 自动降低 6–9 dB；台词结束后 300–500 ms 恢复。
- 场景切换用 0.8–1.5 秒交叉淡化，不硬切。
- 关键台词前允许 1.5–3 秒完全静默。
- 手机震动、门铃等高频音效不连续轰炸，避免耳机玩家疲劳。

## 4. 原创 BGM 设计

只制作 5 首主素材，通过减轨、滤波、速度和混响变化覆盖全游戏。这样既能统一主题，也能控制制作量。

| 编号 | 暂定曲名 | 速度/色彩 | 编制 | 用途 | 循环长度 |
|---|---|---|---|---|---:|
| M01 | **花苞 / Rosebud** | 68 BPM；暖色但不完全落稳 | 毡音钢琴 + 空气 Pad | 标题、序章、“未来的小房子”、回看开场 | 70–90 秒 |
| M02 | **并肩的夜** | 76 BPM；明亮、轻微忙碌感 | 钢琴短音 + 柔和拨弦 + 极轻节拍 | 黑客松、挑衣服、谈《非暴力沟通》 | 60–75 秒 |
| M03 | **蓝光之间** | 60 BPM；冷、克制、有悬而未决感 | 低频脉冲 + 玻璃质感 Pad + 单音钢琴 | 查手机、争执后的冷却 | 60–75 秒 |
| M04 | **隔着半步** | 52 BPM；稀疏、呼吸很长 | 低音 Pad + 独奏钢琴，几乎无鼓 | 发烧夜、分手夜 | 75–100 秒 |
| M05 | **玫瑰开了** | 72 BPM；不狂喜，是真正站稳 | M01 同一音色，增加温暖低音与少量泛音 | 半年后、结尾、制作人员名单 | 80–100 秒 |

### 4.1 回看段变奏

回看不是新写六首歌，而是把对应主曲做成“记忆版本”：

- 削掉鼓与低频，只剩钢琴、呼吸 Pad 和房间底噪。
- 高频轻微衰减，像隔着一层旧照片。
- 点开“没看见的伸手”时，播放统一的 1.2 秒“花瓣提示音”。
- “接住”完成后恢复完整频段，让声音从记忆重新回到现在。
- 不建议整首倒放；倒放容易落入悬疑/恐怖语义，只在转场尾音使用极短反向混响。

### 4.2 原创制作要求

推荐直接为本项目创作，不模仿具体歌手、作曲家、影视或游戏作品。每首保留：

- 工程文件、MIDI、分轨与导出母带；
- 创作日期、作者/生成工具、所用音源名称与音源许可；
- 若使用 AI 音乐工具，保留完整提示词、输出记录和当日服务条款截图；提示词不得出现具体在世音乐人或作品名；
- 上线前使用至少两个音乐识别服务做一次近似曲目排查；命中明显相似结果就重做旋律。

如果时间只够做最小版本，优先完成 M01、M03、M04、M05；M02 可由 M01 加轻节拍和拨弦改编。

## 5. 逐幕声音脚本

| 页面/幕 | BGM 与变化 | 环境音 | 关键声音演出 |
|---|---|---|---|
| 标题/序章 | M01，只有钢琴与 Pad | 很轻的风与纸张摩擦 | 点击“开始”时落下一粒花瓣提示音，不做传统游戏确认声 |
| 幕 1：叫花鸡 | 会场用 M02；转楼梯间后减去节拍，只留钢琴 | 键盘、空调、远处人声；楼梯脚步和包装盒 | Sean 第四次说“等一下”后把环境声稍抬高，表现 Vera 被忽略；他说“谢谢你能跟我说”时音乐第一次落稳 |
| 幕 2：挑衣服 | M02 的轻快版，拨弦更明显 | 衣架轻碰、远处商场底噪 | 试衣帘合上时用柔和布料声；不要加喜剧“换装”音效 |
| 幕 3：非暴力沟通 | M01 的完整暖版 | 校园风、树叶、偶尔远处单车 | 谈到“以后的小房子”时让玫瑰动机第一次完整出现，但结尾仍留一个未解决尾音 |
| 幕 4：关注列表 | 开始无旋律，仅手机电流感；翻到关注列表时进入 M03 | 手机震动、指尖滑动、室内钟声 | Vera 质问前音乐瞬间抽空；争执期间不加鼓，避免把冲突英雄化；Sean 关上后只剩低频底噪 |
| 幕 5：外卖粥 | M04，左右声场轻微分开 | 左侧宿舍雨/空调，右侧便利店冰柜/灯管 | 两地环境保持不同，但对白永远居中；“工作也比家庭重要吗”后静默约 2 秒；电话结束后只留冰柜声 |
| 幕 6：好天气 | M04 极简版，前半段可以完全无音乐 | 校园远声、塑料袋、门口风 | “别走 / 再试试”之后进入单音钢琴；真正决定分开时不要上大弦乐；袋子被接过去的摩擦声是本幕最清楚的音效 |
| 幕 7：半年后 | 便利店底噪先出现，再缓慢进入 M05 | 冰柜、门铃、冬夜风 | Vera 说“要，谢谢，今天有点冷”时 BGM 再降 8 dB；说完整后，玫瑰动机第一次真正解决 |
| 回看 | 对应场景的记忆变奏 | 环境音低通、变远 | 每次揭示用同一枚花瓣提示音；“接住”后恢复现场宽度和高频 |
| 结尾/名单 | M05 完整版 | 可无环境音 | rose-bud 变 rose-bloom 时加入一次柔和泛音，不做胜利音效 |

### 5.1 建议制作的最小音效包

只需要 12 类，足以让场景活起来：

1. 键盘连续输入（3 个循环变体）
2. 空调/电脑风扇房间底噪
3. 楼梯脚步
4. 外卖盒与筷子
5. 衣架/布料/试衣帘
6. 校园风与树叶
7. 手机轻震（2 个强度）
8. 手机滑动/锁屏
9. 宿舍雨夜/窗边风
10. 便利店冰柜与灯管底噪
11. 便利店门铃（自行合成，避免直接用知名连锁店提示音）
12. 塑料袋/装衣物的小袋子

能自己录的尽量自己录。手机、键盘、衣架、包装盒、塑料袋都可以在安静房间用手机完成，之后只做降噪、裁切和响度统一。

## 6. 角色配音方案

### 6.1 配音范围

推荐三级优先级：

| 优先级 | 必配内容 | 理由 |
|---|---|---|
| P0 | Vera/Sean 所有说出口的对白、玩家选择后的实际台词、每幕第一句与最后一句旁白 | 角色关系完整，比赛 Demo 最直观 |
| P1 | 回看中的藏话、接住回应、所有关键转折旁白 | 强化核心玩法“看见” |
| P2 | 普通动作描写与所有选择引导语 | 只在时间和 TTS 表现足够好时加入 |

现有脚本默认收集全部 383 句。落地时建议给 `Moment` 增加 `voice?: "full" | "key" | false` 或建立单独白名单，让普通动作旁白不自动朗读。

### 6.2 三个声音的人设

#### Vera

- 年龄感：20–23 岁，不幼态，不“甜妹营业”。
- 音色：清楚、靠前、略偏明亮，但有克制感。
- 日常：语速稍快，句尾有行动力；强势是有主见，不是娇蛮。
- 焦虑：呼吸变浅、语速加快，重音落在事实词上；禁止演成歇斯底里。
- 分手：比平时慢 10%–15%，有停顿但不持续哭腔。
- 半年后：声线不必变成熟，只需要句子说得完整、句尾不再吞掉。

**首选试音**：腾讯云当前官方表中的 `601009 爱小芊（聊天女声，多情感）`。  
**备选**：`603007 邻家女孩（聊天女声）`；若预算/接口兼容优先，也可保留现有 `101001 智瑜`。

#### Sean

- 年龄感：21–24 岁，学生，不要播音腔和“霸总低音”。
- 音色：温和、略低、带一点疲惫的空气感。
- 日常：话不多，反应自然，不故作深沉。
- 过载：不是冷酷，而是注意力被任务占满；句子短、轻、没能抬头。
- 防御：音量不升高，语速反而慢下来，像把门关上。
- 挽留：第一次允许气息不稳，但不能变成戏剧化哭喊。

**首选试音**：`601008 爱小豪（聊天男声，多情感）`。  
**备选**：`602004 暖心阿灿（聊天男声）` 或 `601014 爱小简（聊天男声）`。

#### 旁白

- 不是上帝视角，也不是情感电台；更像几年后的自己，平静地碰一碰旧照片。
- 速度比角色慢约 8%，停顿清楚，音量略低。
- 不替人物判断对错，金句也不要“念金句”。
- 回看时可用同一音色，但减少混响、拉近距离，表现“终于看见”。

**首选试音**：`501002 智菊（阅读女声）`。  
**备选**：`601013 爱小伊（阅读女声）`。  
如果旁白与 Vera 太接近，可改用克制的男声阅读音色，但不要使用 Sean 同款。

### 6.3 情绪映射

若使用支持多情感的 `601008/601009`，不要逐句自由发挥，统一映射：

| 剧情状态 | Vera | Sean |
|---|---|---|
| 甜蜜日常 | 中性为主，关键回应用高兴但降低强度 | 中性，偶尔高兴 |
| 黑客松疲惫 | 中性/平静 | 平静，语速略慢 |
| 查手机争执 | 生气只用于 1–2 个峰值句，其余中性快速 | 平静或中性，不用生气对轰 |
| 发烧夜 | 平静、句尾收紧 | 悲伤低强度，不演虚弱病号腔 |
| 分手 | 悲伤低强度 | 悲伤低强度 |
| 半年后 | 平静 | 不出场 |
| 回看藏话 | 平静、近讲 | 平静、近讲 |

### 6.4 首轮 12 句试音

批量生成前只做以下试音，确认音色、断句、英文和情绪：

- Vera：暖、强势、质问、分手、半年后各 1 句，共 5 句。
- Sean：专注、温柔、防御、疲惫、挽留各 1 句，共 5 句。
- 旁白：暖场与回看各 1 句，共 2 句。

试听时重点检查：

- `Vera`、`Sean`、`Vercel`、`push`、`review` 等中英文混读是否自然；必要时在“显示文本”和“送入 TTS 的文本”之间建立发音替换表。
- 省略号、破折号与括号有没有制造过长停顿。
- 多音字、人名、书名和口语“嗯”“啊”是否稳定。
- 三个声音是否一耳朵可区分，是否符合大学生年龄感。

### 6.5 不采用真人音色克隆

本版本不克隆队员、朋友、演员、网红或明星的声音。通用官方音色已经够用，也省去声纹授权、撤回授权、肖像/人格权益和后续用途争议。若未来改用真人演员，必须签署覆盖“游戏内、宣传视频、比赛展示、长期在线”的书面授权，并明确是否允许 AI 训练或声音复刻；未明确即视为不允许。

## 7. 混音与文件标准

### 7.1 建议标准

| 资产 | 格式 | 采样率/声道 | 建议响度 | 峰值 |
|---|---|---|---|---|
| BGM | OGG/Opus（当前生成器） | 48 kHz / Stereo | 约 -22 至 -20 LUFS（母带） | ≤ -1 dBTP |
| 角色语音 | MP3 或 OGG | 24 kHz / Mono；不支持则 16 kHz | 约 -18 至 -16 LUFS | ≤ -1 dBTP |
| 环境音 | OGG/Opus（当前生成器） | 48 kHz / Stereo | 约 -28 至 -24 LUFS | ≤ -3 dBTP |
| 瞬态音效 | OGG/Opus（当前生成器） | 48 kHz / Mono 或 Stereo | 以不刺耳为准 | ≤ -3 dBTP |

页面实际播放时，BGM 默认比对白低约 8–12 dB；不要只依赖每个文件各自的响度，客户端仍应做对白 Ducking。

### 7.2 文件结构

```text
public/audio/
├── bgm/
│   ├── m01-rosebud.ogg
│   ├── m02-side-by-side.ogg
│   ├── m03-blue-light.ogg
│   ├── m04-half-step.ogg
│   ├── m05-bloom.ogg
│   └── memory/...
├── ambience/
│   ├── hackathon-room.ogg
│   ├── campus-wind.ogg
│   ├── dorm-rain.ogg
│   └── konbini-hum.ogg
├── sfx/
│   ├── rose-reveal.ogg
│   ├── phone-vibrate-soft.ogg
│   └── ...
├── voice/
│   └── <hash>.mp3
└── manifest.json
```

现有代码把语音直接放在 `public/audio/<hash>.mp3`。第一版可以保持兼容；等 BGM 接入时再把语音迁移到 `voice/`，同时修改 `use-voice.ts` 的路径。

### 7.3 播放交互

- 浏览器自动播放受限，首页必须在用户点击“开始”后再启动声音。
- 设置中分成“音乐、语音、音效”三个独立开关，不只提供总静音。
- 字幕永远显示，关闭配音不影响阅读。
- 切到后台时暂停 BGM；回到前台时淡入，不从头重播。
- 玩家快速翻页时立即停止上一句，不让多句重叠。
- 对长句提供“再次播放”，但不强制玩家等语音念完才能继续。

## 8. 版权与合规策略

“不侵权”不能只靠网站写着“免费”来判断，真正需要的是**来源清楚、许可匹配、证据可追溯**。

### 8.1 资产来源优先级

1. **团队原创并保留工程文件**：BGM 首选。
2. **团队自行录制/合成**：日常环境音和交互音首选。
3. **CC0 素材**：环境底噪的应急来源；即使法律上不要求，也建议在鸣谢页署名。
4. **明确允许商业使用的内容许可素材**：只作为备选，下载时保存许可证据。
5. **不使用**：音乐平台下载、影视/动漫/其他游戏音频、短视频热门原声、来源不明素材包、仅写“仅供学习交流”的文件。

### 8.2 可用的应急素材来源

#### Freesound

只筛选 **CC0**。Freesound 官方说明同时提供 CC0、CC BY、CC BY-NC 等不同许可；CC BY 需要署名，CC BY-NC 不适合可能商业化或参赛宣传的项目。平台也提醒素材由用户上传，仍可能存在误传，因此必须保留原始页面和许可记录。

- 许可说明：<https://freesound.org/help/faq/>
- 本项目原则：P0 只用 CC0，不混用 BY-NC。

#### Pixabay

仅在原创音乐来不及完成时作为 BGM 备选。Pixabay 当前内容许可允许商业和非商业使用、复制、修改或改编，但禁止把内容以“独立素材”的形式再分发；同时平台明确不保证每份用户上传内容都已取得所有第三方许可。因此，游戏内嵌使用通常符合其内容许可思路，但仍要做来源留档和相似曲目排查。

- 许可摘要：<https://pixabay.com/service/license-summary/>
- 完整条款：<https://pixabay.com/service/terms/>
- 下载后必须记录具体作品页、作者、下载日期、许可版本与文件 SHA-256。

### 8.3 腾讯云 TTS

腾讯云官方基础语音合成接口明确面向 App、智能设备、语音播报等应用场景；本项目继续走服务端离线生成，不把 SecretKey 暴露到浏览器。音色和参数以生成当天官方文档/控制台为准。

## 9. 技术落地建议

### 9.1 剧本数据增加声音字段

```ts
interface SceneAudio {
  bgm: string;
  ambience?: string;
  bgmVariant?: "full" | "memory" | "minimal";
  bgmGain?: number;
}

interface Scene {
  // 现有字段...
  audio?: SceneAudio;
}

type Moment =
  | { kind: "narr"; text: string; voice?: "key" | "full" | false }
  | { kind: "line"; who: "vera" | "sean"; text: string; emotion?: string }
  | { kind: "sfx"; src: string; gain?: number }
  // 其余现有类型...
```

声音字段与剧本同源，场景切换时自动切音乐，台词情绪也能进入 TTS 生成器，不再把所有情绪都挤在一个全局音色参数里。

### 9.2 增加统一 Audio Manager

建议新增 `lib/use-audio.ts`，统一管理：

- BGM 循环、交叉淡化和页面可见性；
- 环境音循环；
- SFX 并发数量；
- 语音播放与 BGM Ducking；
- 音乐/语音/音效三组音量和持久化；
- 首次用户手势后解锁 AudioContext。

不要让每个页面各自创建多个不受控的 `<audio>`；场景切换和快速点击时很容易残留声音。

当前原创母带与音效可用 `pnpm gen:audio` 重建；生成器需要本机安装 `ffmpeg`，输出为 48 kHz Stereo Ogg/Opus。资产哈希及来源记录见 `docs/audio-asset-register.csv`。

### 9.3 TTS 生成器需要的小改动

1. 先更新/确认 `VOICE_TYPES`，不要直接沿用未验证的 `101009`、`101018`。
2. 把 `SampleRate` 从 16 kHz 提升到所选音色支持的 24 kHz；若接口拒绝则自动回退 16 kHz。
3. 对支持情绪的音色传入 `EmotionCategory`，并用白名单限制可选情绪。
4. 增加 `ttsText` 或发音词典，解决中英混读，不改屏幕显示文案。
5. `--dry` 输出 P0/P1/P2 数量和预计字符数。
6. 生成失败写入失败清单并支持重试；不要简单拼接多个 MP3 字节作为长句，长句应分片后用音频工具无损/重编码合并，避免部分播放器时间轴异常。

## 11. 验收标准

达到以下条件才算声音方案真正完成：

- 玩家闭眼也能分辨 Vera、Sean 和旁白。
- 暖、冷、回看、治愈四个阶段的声音颜色明显，但听起来仍属于同一部作品。
- 对白任何时候都清楚，不需要调高系统音量才能听见。
- 分手段最痛的时刻来自表演和留白，不来自突然变大的音乐。
- 关闭任一声音类别后，游戏仍可完整游玩。
- 快速翻页、切后台、切场景不会叠音或留下幽灵循环。
- 所有音频都有来源、许可和哈希记录；没有“我记得这个应该能用”的资产。

## 12. 最终推荐配置

如果现在就要定案，推荐：

- **音乐**：5 首项目原创器乐曲，主音色为毡音钢琴 + 柔和 Pad；不使用外部成品 BGM。
- **环境音**：日常物件自行录制；无法录制的底噪只从 Freesound CC0 补齐。
- **Vera**：优先试听 `601009 爱小芊`，备选 `603007 邻家女孩`。
- **Sean**：优先试听 `601008 爱小豪`，备选 `602004 暖心阿灿`。
- **旁白**：优先试听 `501002 智菊`，备选 `601013 爱小伊`。
- **配音范围**：角色全配、关键旁白配、普通叙述不配。
- **核心演出**：查手机前抽掉音乐；发烧夜用左右环境声制造距离；分手关键句后静默；半年后让未解决的玫瑰动机第一次落稳。

这套配置既符合项目现有技术栈，也最贴合《玫瑰无限》“没有坏人、先错过、再看见、最后各自长大”的气质。
