# 《Rose Infinity · 玫瑰无限》音乐、环境音与音效方案

> 版本：v2.0  
> 更新日期：2026-07-15  
> 适用范围：网页游戏 Demo、比赛录屏与公开发布

## 1. 最终决定

本作不使用角色配音或旁白朗读，声音只保留三层：

1. 原创现代古典/新古典钢琴配乐；
2. 场景环境音；
3. 少量叙事与交互音效。

文字始终是叙事主体。音乐负责托住情绪，环境音建立地点，音效只标记玩家真正需要注意的动作。取消配音可以避免合成腔、地域口音和表演不一致破坏代入感，也减少第三方声音授权风险。

## 2. 音乐方向

### 2.1 气质参考的转译

用户给出的气质参考是 Olivia Belli 的《Intercosmia, Vol. 2》。公开资料显示该 EP 包含两首独奏钢琴、一首独奏大提琴和一首弦乐三重奏；相关官方介绍强调本质、内省与精神性。

本项目只提取以下抽象特征：

- 现代古典/新古典钢琴；
- 冥想、空灵、内省；
- 低密度音符与大面积留白；
- 慢速、开放和弦、非规律呼吸；
- 长尾空间混响；
- 极少量低音弦乐或泛音支撑。

不会复制参考作品的旋律、主题、和声进行、配器段落或录音质感，也不会把艺术家或专辑名写入音乐生成提示词。所有正式母带均由项目脚本从零合成，不含参考音乐采样。

资料：

- <https://classical.music.apple.com/us/album/1793368578?l=en-US>
- <https://www.fabermusic.com/music/intercosmia-vol-2>
- <https://www.xximrecords.com/releases/intercosmia-vol-1/>

### 2.2 音乐语言

- 主奏：毡音感钢琴；音头柔和，但不能糊成 Pad。
- 和声：开放五度、add2/add6、悬置音；减少功能和声式强终止。
- 节奏：44–76 BPM；整体避免固定鼓点和连续八分音符琶音，M02 例外保留轻微拨弦脉冲以表现会场忙碌感。
- 旋律：短小、原创、可辨认但不过度重复。
- 空间：中长混响，保留近距离琴槌触键感。
- 动态：整体克制；冲突场景靠抽空音符，不靠突然放大。
- 禁止：短视频催泪套路、宏大弦乐、流行鼓组、歌词、人声哼唱、模仿具体作品。

## 3. 五首原创配乐

| 编号 | 曲名 | 速度 | 音乐角色 | 主要使用场景 |
|---|---|---:|---|---|
| M01 | 花苞 / Rosebud | 52 BPM | 温暖但未完全落稳；疏朗钢琴 | 标题、序章、校园长椅、回看开场 |
| M02 | 并肩的夜 / Side by Side | 76 BPM | 已确认的原版母带；轻拨弦脉冲与柔和钢琴，五首中最有流动感 | 黑客松、挑衣服 |
| M03 | 蓝光之间 / Blue Light | 48 BPM | 冷色开放和弦、稀薄低音弦乐 | 查手机、关系转冷 |
| M04 | 隔着半步 / Half Step | 52 BPM | 已确认的原版母带；最少音符、最长留白 | 发烧夜、分手夜 |
| M05 | 玫瑰开了 / Bloom | 56 BPM | 同一主题终于站稳，不做胜利音乐 | 半年后、最终结局 |

所有曲目为 51–60 秒无缝循环。项目默认播放音量约 0.18；回看段降至约 0.11，使音乐像隔着记忆传来。

## 4. 环境音

| 文件 | 场景 | 使用原则 |
|---|---|---|
| `campus-wind.ogg` | 校园、分手日 | 风与空气流动；晴天也保留一点距离感 |
| `konbini-hum.ogg` | 便利店 | 冰柜与灯管的稳定嗡鸣；衬托半年后的平静 |

黑客松和宿舍场景不使用环境音，只让配乐与文字建立空间。其余环境音默认音量约 0.065，回看段约 0.035；全部为程序合成，不使用外部录音。

## 5. 音效

| 文件 | 用途 | 触发点 |
|---|---|---|
| `rose-reveal.ogg` | 花瓣/看见提示 | 回看中揭示隐藏心情、完成“接住” |
| `phone-vibrate-soft.ogg` | 手机轻震 | 手机聊天演出开启 |
| `phone-lock.ogg` | 锁屏 | 后续需要明确关手机时使用 |
| `konbini-door.ogg` | 原创两音门铃 | 进入半年后便利店 |
| `ui-soft-tap.ogg` | 柔和确认声 | 开始游戏、选择选项、结局按钮 |

音效不为每次翻页播放。重复点击音会让阅读变成操作界面，削弱叙事留白。

## 6. 逐幕声音脚本

| 页面/幕 | BGM | 环境音 | 关键音效与留白 |
|---|---|---|---|
| 标题/序章 | M01 | 无 | 点击开始播放一次 soft tap；用户手势同时解锁浏览器音频 |
| 叫花鸡 | M02 | 无 | 由 M02 的流动感表现会场忙碌，角色冲突处不加重音乐 |
| 挑衣服 | M02 | 无 | 只保留选择确认声 |
| 非暴力沟通 | M01 | 校园风 | 关于未来的段落让钢琴自然流动，不额外煽情 |
| 关注列表 | M03 | 无 | 手机演出开启时轻震一次；争执靠冷和声与留白 |
| 外卖粥 | M04 | 无 | 远距离感来自音乐留白，不用语音左右分屏 |
| 好天气 | M04 | 校园风 | 分开决定后不要追加煽情音效 |
| 半年后 | M05 | 便利店嗡鸣 | 进场门铃一次；音乐不做“圆满胜利” |
| 回看 | 对应场景 BGM 低音量版 | 对应环境音低音量版 | 每次揭示使用统一花瓣提示音 |
| 结局 | 未回看完成用 M04；完成后用 M05 | 无 | 按钮只使用 soft tap |

## 7. 播放与交互规范

- 全局声音管理器挂在根布局，跨页面导航时保持状态。
- BGM 与环境音切换使用约 0.7–0.9 秒淡出/淡入。
- 浏览器首次点击或按键后才开始播放，符合自动播放限制。
- 页面进入后台时暂停循环，回到前台后恢复。
- 只有一个“音乐与音效”总开关，状态保存在 `localStorage`。
- 静音时立即停止正在播放的瞬态音效，并把循环轨淡出。
- 所有游戏文本不依赖声音；静音后仍能完整游玩。
- 默认建议戴耳机，但外放时也不能出现刺耳高频或过强低频。

## 8. 文件与实现

```text
public/audio/
├── bgm/
│   ├── m01-rosebud.ogg
│   ├── m02-side-by-side.ogg
│   ├── m03-blue-light.ogg
│   ├── m04-half-step.ogg
│   └── m05-bloom.ogg
├── ambience/
│   ├── campus-wind.ogg
│   └── konbini-hum.ogg
├── sfx/
│   └── ...
└── soundtrack.json
```

关键代码：

- `scripts/gen-audio-assets.mjs`：确定性生成原创音乐、环境音和音效；M02、M04 作为已确认的项目原版母带被保留，不在重跑时覆盖；
- `components/soundscape-provider.tsx`：循环、淡化、静音、后台暂停、SFX 播放；
- `lib/audio.ts`：资产路径、场景到声音的映射；
- `docs/audio-asset-register.csv`：来源、许可与 SHA-256 记录。

重新生成：

```bash
pnpm gen:audio
```

需要本机安装 `ffmpeg`。输出为 48 kHz Stereo Ogg/Opus。

## 9. 版权与合规

- 正式文件均为项目自产程序合成，不包含第三方采样；M02、M04 保留已确认的原版项目母带，其余文件由当前脚本确定性重建。
- 不下载、不剪辑、不变速参考专辑中的任何录音。
- 不按参考作品逐段重建旋律、和声或配器。
- 保留生成脚本、生成日期、资产清单和哈希，形成可追溯证据链。
- 若未来引入外部录音，只接受团队原创或许可明确适用于商业游戏分发的素材。
- 不使用“仅供学习”“仅个人使用”或来源不明的音乐与音效包。

## 10. 验收标准

- 暖、冷、分手、治愈四个阶段可以靠音色和留白分辨，但仍属于同一套音乐语言。
- 钢琴听起来安静、内省、空灵，不像节奏游戏或短视频配乐。
- 循环点无爆音、明显断点或突然换和弦。
- 环境音可感知但不抢文字；连续佩戴耳机十分钟不疲劳。
- 快速切页、切后台和反复静音不会叠音或留下幽灵循环。
- 所有音频都能由脚本重新生成，登记表哈希与正式文件一致。
