# 《Rose Infinity》CodeBuddy 证据摘要

> 这份摘要帮助评委快速定位腾讯工具的实际使用记录；完整对话仍以同目录下的会话文件为准。

## 一、可以直接核验的事实

### 1. CodeBuddy 是核心开发工具

- 本项目保留 12 个有效 CodeBuddy 会话；
- 共 785 轮人机对话、1183 次工具调用；
- 记录覆盖需求拆解、代码生成、文件编辑、调试、故事实装、Prompt 设计、AI 生图和部署准备；
- 时间范围为 2026-07-02 至 2026-07-14，并可与 Git 提交历史相互印证。

### 2. 确实使用了腾讯混元 Image V3

会话 [`25ae0c09-f8f0-4c8d-8877-31f6d18fee50`](./25ae0c09-f8f0-4c8d-8877-31f6d18fee50.md) 明确记录：

1. 在 CodeBuddy 中执行 `/model:text-to-image`；
2. 输出显示生图模型切换为 `hunyuan-image-v3.0`；
3. 随后加载 `ImageGen` 工具；
4. 生成 Vera / Sean 角色表情与水彩场景，并继续进行风格统一、去水印和资产落盘。

因此可以准确表述为：**项目通过 CodeBuddy 的 AI 生图工作流使用腾讯混元 Image V3 参与游戏美术资产开发。**

### 3. 腾讯混元聊天接口完成了工程接入

会话 [`ce7d3415-2435-49d6-9b91-d2dd9a2ab861`](./ce7d3415-2435-49d6-9b91-d2dd9a2ab861.md) 包含混元客户端、`/api/npc`、`/api/reveal`、Prompt 和接口调试记录。

需要保持准确：历史记录中的混元聊天测试包含 API Key 配置失败的 401 过程；当前比赛版的正式分支使用作者校准文本，不依赖混元聊天接口完成主线。因此它应描述为“已实现并带兜底的生成式扩展”，不应描述为当前核心剧情由混元实时驱动。

## 三、比赛材料中的推荐表述

> 《Rose Infinity》使用腾讯云 CodeBuddy 进行核心开发，保留 12 个有效开发会话、785 轮对话和 1183 次工具调用。游戏美术开发过程中，通过 CodeBuddy 的 ImageGen 工作流实际使用腾讯混元 Image V3 生成角色与场景资产。项目同时完成腾讯混元 NPC 与内心揭示接口及失败兜底；竞赛版正式剧情采用作者校准文本，因此不把接口夸大为当前核心玩法。

## 四、不建议使用的表述

- “当前全部剧情由混元实时生成”；
- “腾讯云 TTS 构成最终配音模块”（最终版本已取消配音）；
- “所有声音均由腾讯工具生成”（正式声音为项目脚本程序合成）；
- “所有 42 张视觉都能逐张证明由同一个腾讯模型生成”（记录能证明混元 Image V3 工作流，但不应超出日志逐张推断）。

## 五、安全处理

- 导出器默认只扫描当前项目，不会静默混入其他仓库；
- API Key、SecretId、SecretKey、Bearer Token 与 URL 查询密钥自动脱敏；
- 本机主目录替换为 `~`；
- 内部 reasoning 与文件快照不进入提交材料；
- 导出后已用当前 `.env.local` 的实际配置值反向扫描，敏感值命中数为 0。
